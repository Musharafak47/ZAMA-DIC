Zama / FHE Glossary

Bootstrapping

Category: Mechanics

The crucial process in Fully Homomorphic Encryption that reduces the "noise" inside a ciphertext. It refreshes the encrypted data so that more calculations can be performed on it without corrupting the result.

Ciphertext

Category: General

Data that has been encrypted. In FHE, this data looks like random noise to the server, but the server can still perform math on it.

Concrete

Category: Zama Product

Zamaâ€™s open-source framework and compiler. It converts high-level programming code (like Python) into low-level FHE circuits, handling the complex cryptography automatically.

Concrete ML

Category: Zama Product

A specialized library built on top of Concrete designed for Data Scientists. It allows users to turn standard Machine Learning models (like Linear Regression or Decision Trees) into FHE-compatible models.

End-to-End Encryption (E2EE)

Category: General

A standard security method where data is encrypted during transfer. Unlike FHE, E2EE requires data to be decrypted before it can be processed or analyzed.

Evaluation Key

Category: Keys

Also known as the "Server Key." A public key given to the cloud provider that allows them to compute on the ciphertext without being able to decrypt or read the data.

FHE (Fully Homomorphic Encryption)

Category: General

A form of encryption that allows computations to be performed on encrypted data. The result of the computation, once decrypted, matches the result as if the operations had been performed on the plain text.

fhEVM

Category: Zama Product

A protocol developed by Zama that integrates FHE into the Ethereum Virtual Machine. It enables confidential smart contracts on public blockchains.

LWE (Learning With Errors)

Category: Math/Security

The difficult mathematical problem that forms the security foundation of the TFHE scheme. It involves solving linear equations that have had small errors introduced to them.

Noise

Category: Mechanics

A small amount of randomness added to data during encryption to ensure security. If noise grows too large during calculations, the data becomes unreadable; Bootstrapping is used to fix this.

PBS (Programmable Bootstrapping)

Category: Mechanics

A unique feature of the TFHE scheme. It cleans the noise from the ciphertext while simultaneously applying a function (like a lookup table) to the data. This makes Zama's approach very fast for non-linear operations (like Neural Networks).

Plaintext

Category: General

The original, unencrypted input data (e.g., a credit card number or health record) visible only to the user who holds the Secret Key.

Secret Key

Category: Keys

Also known as the "Client Key." The private key held exclusively by the user. It is the only key capable of encrypting the original data and decrypting the final result.

TFHE (Torus Fully Homomorphic Encryption)

Category: Cryptography Scheme

The specific FHE algorithm that Zama uses. It is known for its speed, exact results (no approximation), and efficient bootstrapping capabilities.

TFHE-rs

Category: Zama Product

The pure Rust implementation of the TFHE scheme. It is the low-level cryptographic engine that powers Zama's higher-level tools like Concrete.

Zama

Category: Company

An open-source cryptography company building tools to make Fully Homomorphic Encryption accessible and fast for developers.
